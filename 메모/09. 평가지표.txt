[회귀] : sklearn - r2, mean_squared_error
[분류] : sklearn - accuracy, AUC, F1-Score

train데이터로 학습해서 만든 모델의 평가지표는 높게 나오는데 대회에 제출하면 점수가 더 낮게 나올경우 그 이유 :
대회에서 점수산정할때 private data로 했기때문. private 데이터로는 학습이 안되어서 모델이 특성을 반영못했기 때문이다.

[분류] : 다중분류안에 이진부류가 포함됨 (2x2혼동행렬이 이진분류)
1. accuracy : 데이터불균형(한쪽 클래스에 데이터수가 치우침)에 민감
2. AUC : 다양한 임계치에 따른 ROC Curve의 면적계산. 데이터불균형에 덜 민감 (타겟 데이터 : 0 또는 1 / 테스트 데이터 : 0.954, 0.453 같은 연속데이터)
    - 테스트 데이터가 이진데이터일때 AUC를 안쓰는 이유 : ROC커브를 그리면 일직선형태가 나와서 면적계산의미가 없다.
3. F1-Score :  혼동행렬에서 recall과 precision의 조화평균(작은값에 더 가까워서 데이터불균형 해소)

데이터불균형이 문제인 이유 : 학습시 특정 클래스에만 과적합된 학습. 특정클래스를 선택했을시 테스트시에만 비정상적인 높은 정확도 보임.
